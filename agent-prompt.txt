# paper2ppt Agent — 格式化与排版补充规则
# 本文件为 prompt-base.txt 的补充，记录了从 v1-v5 迭代中积累的经验教训
# Agent 每次运行时必须同时遵守 prompt-base.txt 和本文件的规则

## 模板保护规则（最高优先级）

1. 封面、目录、章节页、致谢页：保留模板所有原始形状和设计元素，仅修改文字
2. 致谢页：完全不可修改，保持模板原文
3. 章节页：仅修改编号（如01→02）和标题文字，不修改任何图形
4. 目录页：通过XML级别的<a:t>元素文本替换修改条目，保留所有原始图形
5. 封面：仅修改标题文字和汇报人信息，保留所有装饰元素

## 内容页格式规则

6. 标题：24pt Times New Roman 粗体 #007191 青色
7. 正文：全部统一16pt Times New Roman
8. 每页3行正文，每行15-20字（中文），不含图片编号引用
9. 每行文字放在独立文本框中，前面加 #007191 青色小圆点标记（直径0.18英寸）
10. 行间距：3行时0.9英寸，更多行时动态调整

## 图片规则

11. 宽度统一5.5英寸，高度按原比例自动计算
12. 统一放置在页面左侧（x=0.3英寸），右侧放文字
13. 2张图片时垂直堆叠（y≈1.05, 3.85），3张时紧凑排列（y≈1.05, 2.8, 4.7）
14. 文件名格式：{FigureNum}{Panels}.jpg（如1A.jpg, 1DEF.jpg, 5ABCD.jpg）

## 语言规则

15. 所有内容使用中文，首次出现的英文专业术语加括号标注中文翻译
16. 不直接粘贴论文英文原文片段
17. 用自己的话总结论文发现，不引用图表编号（如"见图1A"）

## 视觉页设计规则

18. 作者团队页：使用色块卡片布局（左=期刊信息，右=机构，底=通讯作者+研究基础）
19. 课题背景页：三色卡片横向排列 + 底部假说框
20. 结果总结页：横向流程箭头图（每个步骤一个圆角色块）+ 三维度证据卡片 + 结论文字
21. 讨论页1：编号圆圈 + 标题 + 详述 + 分隔线的列表布局
22. 讨论页2：左右双栏（局限性 | 临床意义），不同底色区分

## 颜色规范

23. 主色：#007191（青色），用于标题、标记、重点元素
24. 辅助色：#1A2E4A（深海军蓝）、#D94F4F（红色强调）、#2E7D32（绿色）
25. 浅色背景：#E0F0F5（浅青）、#E8F5E9（浅绿）、#FDEDEC（浅红）
26. 正文色：#333333，装饰灰：#A5A5A5

## 技术陷阱（避免重蹈）

27. 幻灯片克隆时图片引用会丢失 → 不克隆模板幻灯片，改为从头构建内容页
28. 模板幻灯片reorder时需操作sldIdLst XML → 使用copy.deepcopy避免引用冲突
29. 封面文字替换需扩大文本框宽度（原2.1in → 4.5in），否则文字被截断
30. 目录文字替换需用XML级别<a:t>遍历，python-pptx的shape遍历无法进入深层GROUP
31. 中文字符检查：不能用'结果分析'匹配模板实际文本'结论分析' → 先dump模板实际<a:t>文本再写替换映射
32. 文件保存时PowerPoint占用会导致PermissionError → 换文件名保存
